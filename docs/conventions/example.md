# Example

Use examples to demonstrate the convention.

## Illustrative examples

For illustrative examples, choose small generic names that are unlikely to be
renamed with local code, such as `app`, `foo_bar`, or `src`.

Do not copy current functions, variables, paths, or release assets just to make
an example feel realistic. That couples the docs to implementation details and
creates doc churn when code is renamed.

## Concrete examples

Use real local names only when the doc is specifying that concrete API, layout,
command, or file path. In that case, keep the example exact and update it with
the code.
