# Inline code

Backticks mark a literal token, not emphasis. Use them when the reader should
read a specific sequence of characters instead of an ordinary prose concept.

A token needs backticks only when one of these is true:

- The bare token could be mistaken for ordinary English: `make`, `all`,
  `include`.

- The sentence is about the token's literal form: exact characters, case,
  punctuation, or filename spelling, as in `Makefile` versus `makefile`.
